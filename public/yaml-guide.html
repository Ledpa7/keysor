keysor.yaml 설정 완벽 가이드

키서(Keysor)는 YAML 형식의 설정 파일을 지원하여 작동 모드, 세부 속도 및 가속율, 기능별 커스텀 단축키 매핑을 자유롭게 구성할 수 있습니다. 본 문서는 각 설정 항목에 대해 누구나 알기 쉽게 나누어 설명합니다.

1. 설정 파일 위치

키서는 실행 시 아래 경로의 폴더를 검색하여 설정 파일을 로드합니다. 파일이 존재하지 않는 경우 자동으로 기본값을 포함한 템플릿을 생성합니다.

기본 로드 경로

Windows 탐색기 경로
C:\Users\<사용자이름>\.keysor\keysor.yaml
단축 환경 변수 경로
%USERPROFILE%\.keysor\keysor.yaml

💡 초간단 실행 팁

바탕화면이나 탐색기에서 일일이 폴더를 찾아 들어갈 필요 없이 아래 단축키 조합으로 한 번에 진입이 가능합니다.

1. Win + R 키를 눌러 실행 창 오픈
2. 실행 창에 %USERPROFILE%\.keysor 입력 후 엔터
* 파일이 없을 시 최초 실행할 때 자동으로 템플릿 파일이 생성됩니다.

2. 주요 설정 항목 설명

설정 파일 최상단의 settings: 필드에서 마우스의 작동 방식을 변경하는 핵심 매개변수들입니다.

modifier_mode String

마우스 모드 활성화 방식을 결정합니다.

• "Toggle" : 키를 탭하여 ON/OFF (기본값)
• "Hold" : 키를 누르고 있는 동안에만 작동
base / max_speed Pro 전용

마우스 속도 범위를 지정합니다.

• base_speed : 시작 속도 (기본 1.5px)
• max_speed : 최대 가속 한계 (기본 30.0px)
acceleration Pro 전용

이동 키를 유지할 때 가속도가 붙는 가속 승수입니다.

• 기본값은 1.5이며, 값이 클수록 마우스 이동 조작 시 빠르게 가속되어 최종 최고 속도에 도달합니다.
magnetic_mode Pro 전용

키서 전용 로컬 HUD 패널 내부의 버튼에 마우스가 접근할 때 자동으로 중심부에 달라붙는 기능입니다. (기본값: false)

global_magnetic_mode Pro 전용

전체 OS 윈도우 창 및 웹 브라우저 내 조작 버튼, 체크박스, 하이퍼링크 등 모든 UI 요소 25px 진입 시 자동으로 커서가 스냅 흡착됩니다. (기본값: false)

unified_space_click Boolean

스페이스바 클릭 융합 모드 활성화 여부입니다. (기본값: true)

• 활성화 시 스페이스바 단독 입력은 좌클릭, 연타 시 더블클릭, 길게 누르면 드래그 시작/종료 토글로 작동합니다.

3. 키 매핑 및 단축키 지정

bindings: 필드에서는 트리거 키와 액션을 매핑하여 사용자가 선호하는 조작 배치를 완성할 수 있습니다.

할당 가능한 마우스 액션

MouseMoveUp / Down / Left / Right 마우스 커서 상/하/좌/우 이동
MouseScrollUp / Down / Left / Right 마우스 휠 스크롤 상/하/좌/우
MouseLeftClick / MouseRightClick 마우스 좌클릭 및 우클릭
MouseDoubleClick 마우스 더블 클릭 실행
RunApp:<Path> 지정한 외부 프로그램 실행

사용 가능한 키보드 명칭

트리거 키(trigger)로 맵핑 가능한 키보드 단독 키와 특수 명칭들입니다.

일반 키: "A" ~ "Z" 알파벳 문자열
방향 키: "Up", "Down", "Left", "Right"
기타 특수 키: "Space", "RightShift", "LeftShift", "Tab", "Escape", "F1" ~ "F12"
📌 참고: 키서 1.1 버전부터는 방향키, 숫자 키, F-키들이 설정 파일에 맵핑되지 않은 경우, 차단되지 않고 OS로 투과(Bypass)되어 정상적인 문서 작업이나 단축키 활용이 가능합니다.

4. RunApp 런처 활용 가이드

단축키 등록 한 번으로 컴퓨터의 앱이나 자주 접속하는 웹 페이지를 즉시 구동하는 백그라운드 런처 기능입니다.

작성법 및 규칙

  • 경로 및 파일명에 공백(띄어쓰기)이 있는 경우에는 반드시 경로를 큰따옴표(")로 감싸고, 전체를 작은따옴표(')로 정의해야 합니다.
  • 환경 변수로 등록된 프로그램(예: Chrome, Notepad, Calc 등)은 절대 경로 없이 실행파일명만 적어도 작동합니다.
  • 웹 브라우저를 띄울 때는 실행 프로그램 뒤에 띄어쓰기로 주소를 인자로 넘겨줄 수 있습니다.

YAML 작성 예시

# 일반 메모장 실행
- trigger: "G"
  action: 'RunApp: "notepad.exe"'
# 크롬 브라우저로 깃허브 바로가기
- trigger: "F1"
  action: 'RunApp: "chrome.exe https://github.com"'

5. 기본 YAML 템플릿

설정 파일을 초기화하거나 직접 파일을 복구하고 싶을 때 아래의 표준 기본 설정 템플릿 코드를 복사해서 붙여넣으십시오.

keysor.yaml 
# keysor.yaml - 키서 단축키 및 동작 설정 파일
settings:
  modifier_mode: "Toggle"        # "Hold" (Caps Lock을 누르는 동안에만 조작) 또는 "Toggle" (켜고 끄기)
  modifier_key: "CapsLock"     # 활성화 단축키 (기본값 CapsLock)

  base_speed: 1.5              # 마우스 시작 속도 (픽셀 단위)
  max_speed: 30.0              # 마우스 최대 속도 (픽셀 단위)
  acceleration: 1.5            # 누르고 있을 때의 가속도 비율

  # 스페이스바 단일 키 클릭 연동 특수 기능 활성화
  unified_space_click: true
  double_click_threshold_ms: 100
  drag_hold_threshold_ms: 250
  refresh_rate_hz: 100         # 이동 주기 주사율 (기본값 100Hz, 원격 접속 시 60으로 낮춤)
  magnetic_mode: false         # 자석 모드 활성화 여부 (HUD 버튼에 마우스가 다가가면 흡착)
  global_magnetic_mode: false  # 전역 UI 자석 모드 활성화 여부
  lang_en: true                # 기본 언어 영어 설정 (true: 영어, false: 한국어)
  license_key: ""              # 프로 라이선스 키 (구입 후 발급받은 키 입력)

bindings:
  # ==========================================
  # 1. 이동 바인딩 (하이브리어드 지원)
  # ==========================================
  # 왼손 단독 제어 (WASD)
  - trigger: "W"
    action: "MouseMoveUp"
  - trigger: "S"
    action: "MouseMoveDown"
  - trigger: "A"
    action: "MouseMoveLeft"
  - trigger: "D"
    action: "MouseMoveRight"

  # ==========================================
  # 2. 보조 클릭 및 스크롤 바인딩
  # ==========================================
  - trigger: "R"
    action: "MouseScrollUp"
  - trigger: "F"
    action: "MouseScrollDown"
  - trigger: "Q"
    action: "MouseScrollLeft"
  - trigger: "E"
    action: "MouseScrollRight"
  - trigger: "G"
    action: "MouseRightClick"
  - trigger: "RightShift"
    action: "MouseRightClick"
